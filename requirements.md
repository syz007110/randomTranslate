# File Translator 需求文档（v1）

## 1. 目标
构建一个独立文件翻译工具，支持 `docx / md / json / txt`，在翻译准确的前提下尽可能保持原文件格式不变。

## 2. 核心需求
1. 支持输入格式：docx、md、json、txt。
2. 支持多语言双向翻译（如中->英、英->日等）。
3. 保留原始结构：
   - docx：段落、run、样式尽量保持；
   - md：标题、列表、代码块、链接结构不破坏；
   - json：key 不变，仅翻译 value；
   - txt：段落边界保持。
4. 引擎支持可切换：LLM + 常规翻译（DeepL/Google 等）。
5. 缓存机制：重复文本不重复消耗 token。
6. 术语库机制：按领域与多语言映射约束译法一致性。
7. 输出翻译报告：包含引擎、命中缓存数、失败段落数、格式校验结果。

## 3. 路由策略（v1）
- docx/md：默认 LLM。
- json/txt：默认常规翻译引擎；术语密集或复杂语境可切 LLM。
- 任一引擎失败时支持 fallback。

## 4. 非功能需求
- 可部署到云服务器（Docker）。
- 支持批量任务与异步处理。
- 记录可审计日志。

## 5. 暂不纳入 v1
- PDF 版式还原翻译。
- 前端复杂管理后台。
